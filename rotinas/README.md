# Mapa de Rotinas (`rotinas/`)

## Propósito
`rotinas/` guarda a **cadência operacional** do segundo cérebro: o que roda, com que frequência, para qual canal, com qual papel e em que estado real está.

## Regra estratégica
Se `context/` guarda verdade durável, `projects/` guarda inteligência de projeto e `skills/` guardam capacidade operacional, `rotinas/` guarda **cadência operacional**.

## Categorias sugeridas
| Categoria | Pasta / exemplo | Papel |
|---|---|---|
| Segundo cérebro | `segundo-cerebro/` | consolidar, auditar e proteger o repositório canônico |
| Comunicação e follow-up | `comunicacao/` | resumos, follow-ups, lembretes, handoffs |
| Operação / backlog | `operacao/` | consolidadores de execução, hygiene de tracker |
| Infra / monitoramento | `infra/` | watchdogs, health reports, alertas |
| Governança | `governanca/` | padrões de publicação, critérios, checklists |

## Como ler esta pasta
1. Comece por `STATUS.md` para leitura executiva do parque atual.
2. Vá para a categoria ou rotina específica.
3. Leia o `README.md` daquele job para ver identidade, frequência, destino e papel.

## Critério de qualidade
Abrir `rotinas/` precisa responder rápido:
- quais rotinas existem
- quais estão ativas
- qual a criticidade de cada uma
- onde entregam
- o que é canônico versus opcional
- qual stack operacional sustenta cada job
