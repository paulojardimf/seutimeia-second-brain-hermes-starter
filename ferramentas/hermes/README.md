# Hermes

## Papel
Hermes é o agente principal que lê contexto, executa, registra, automatiza e interage nos canais conectados.

## O que costuma ser documentado aqui
- perfil/personalidade principal
- skills relevantes
- crons relevantes
- canais conectados
- regras de segurança
- padrões de uso (`/skill`, `/reload-skills`, `/cron`, `/restart`)

## Credenciais e configuração
- `~/.hermes/.env` guarda segredos
- `~/.hermes/config.yaml` guarda configuração
- credenciais nunca entram neste repositório

## Exemplos de decisões importantes
- qual provider/modelo usar
- quais toolsets ficam habilitados
- onde as rotinas entregam
- quais perfis existem e para quê
