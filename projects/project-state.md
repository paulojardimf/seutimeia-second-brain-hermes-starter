# Project State

## Visão macro do portfólio
| Projeto / frente | Estado | Tipo | Fonte de execução | Próximo marco |
|---|---|---|---|---|
| Segundo cérebro da operação | ativo | fundação | Linear / manual | onboarding concluído |
| Implantação em cliente exemplo | ativo | delivery | Linear / ClickUp | mapear ferramentas |
| Frente comercial exemplo | acompanhamento | receita | Notion / CRM | proposta validada |
| Roadmap de automações | backlog | melhoria interna | backlog principal | priorização |

## Regras de governança
- backlog detalhado não mora aqui
- este arquivo existe para responder **o que está vivo, por que importa e qual é o próximo marco**
- se um projeto estiver ativo, ele deve aparecer aqui e ter pasta própria em `active/`
