# active/

Projetos vivos, prioritários ou já em execução.

## O que cada pasta de projeto deveria responder
1. o que este projeto é
2. por que ele importa
3. qual o estado atual
4. quais ferramentas / repositórios participam
5. qual o principal risco
6. qual o próximo marco

## Estrutura sugerida por projeto
- `README.md`
- `repositorios/` quando houver mais de um repo ou espelho técnico
- `documentos/` quando houver artefatos importantes
- `historico/` quando começar a acumular material velho
